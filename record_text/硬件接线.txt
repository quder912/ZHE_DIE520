硬件接线
========

⭐⭐⭐ 关键警告：MADLT15SF 单相 200V 主电源接线 ⭐⭐⭐

   单相 200V 输入时，必须接 L1 + L3（不是 L1 + L2！）
   接错 L1 + L2 的后果：
   - CHARGE 充电指示灯亮
   - 数码管显示正常
   - 无任何报警（d12=Err---）
   - 但驱动器 S-RDY=0
   - 试运转失败
   - 伺服 ON 失败
   - PANATERM 能识别但无法启用伺服
   这是个极其隐蔽的坑！未来调试新驱动器一定先验证电源接线！

   端子台接线（200V 系 A~D 型）：
   L1   ── 单相 200~240V 火线
   L2   ── 不接（断开）
   L3   ── 单相 200~240V 零线
   L1C  ── 控制电源火线（与主电源共用 220V）
   L2C  ── 控制电源零线

   三相 200V 输入时：L1、L2、L3 都接。

1. X2 通信连接器（RS485 / RS232 复用）

   MADLT15SF 上的 X2 端口同时支持 RS485 和 RS232 通信，
   使用哪个物理层由 Pr5.37 决定。

   引脚定义（依据 sx-zsv00015 文档）：

   RS485 模式（Pr5.37=2）：
   引脚号  信号          说明
   ────── ──────────── ──────────────────────
   1       GND          接地（信号地）
   7       RS485-       信号负（B / D-）
   8       RS485+       信号正（A / D+）

   RS232 模式（Pr5.37=1）：
   引脚号  信号          说明
   ────── ──────────── ──────────────────────
   1       GND          接地（信号地）
   3       TXD          发送
   4       RXD          接收

   ⚠️ 之前记录的 "1=SG+, 2=SG-, 3=FG" 来自 a6bm-m01-block-modbus 文档，
   与 sx-zsv00015 文档的引脚定义不一致。实际接线请以驱动器外壳标注或
   ime88 主手册配线图为准。两种文档可能对应不同硬件版本。

   ⚠️ 重要！USB-RS232 线和 USB-RS485 线是不同的！
   - 如果买的是 USB转232 线，需要设 Pr5.37=1（RS232 模式）
   - 如果买的是 USB转485 线，需要设 Pr5.37=2（RS485 模式）
   - 设错协议会导致完全没回应！

2. 推荐的接线方式（RS485 模式，Pr5.37=2）

   PC 端 → USB转RS-485转换器 → RS485差分线 → 驱动器 X2

   USB转RS-485 转换器推荐型号：
   - FTDI USB-RS485-WE-5000-BT
   - 或者基于 FT232RL + MAX485 的模块

   接线细节：
   - 转换器 A(+) → X2 引脚 8 (RS485+)
   - 转换器 B(-) → X2 引脚 7 (RS485-)
   - 信号地 GND  → X2 引脚 1 (GND)
   - 如果通信距离超过 100 米或在强干扰环境，需在驱动器和 PC 端并联 120Ω 终端电阻

3. X4 I/O 连接器

   MADLT15SF 的 X4 为 50 针控制 I/O 接口，关键引脚：

   引脚号  信号         说明
   ────── ─────────── ──────────────────────
   29      SRV-ON      伺服使能（与 COM- 短接 = ON）
   41      COM-        公共负端
   26      STB         Block 动作启动触发
   30      SI7         输入信号 7
   31      SI8         输入信号 8
   42,43   COM+        公共正端

   当通过 Modbus 通信控制时（Pr6.28=1），不需要通过 X4 接线来控制，
   通过线圈 0060h 直接控制伺服使能。

4. 通信参数设置（Pr 参数，依据 sx-zsv00015）

   参数       说明                      默认值        实际值
   ──────── ──────────────────────── ──────────── ──────────────────
   Pr5.37    Modbus 连接设置            0          需改为 1(RS232) 或 2(RS485)
   Pr5.31    轴编号（站号）              1          1（默认即可）
   Pr5.30    RS485 通信波特率            2          2 = 9600 bps
   Pr5.29    RS232 通信波特率            2          2 = 9600 bps
   Pr5.38    Modbus 通信设置             0          0 = Even/1bit（默认即可）
   Pr6.28    特殊功能选择                0          需改为 1 = Modbus 启动 Block

   ⚠️ 之前记录的 Pr5.00/Pr5.01/Pr5.02/Pr5.03 为错误编号，已修正。
   ⚠️ Pr5.04 不要乱改：=2 在某些固件版本反而触发 Err38.0！
      调试通讯不需要改 Pr5.04，保持出厂值 0 即可。

5. 调试踩坑记录（2026-05-26）

   坑1：单相 200V 接 L1+L2 不工作 → 必须 L1+L3
   坑2：USB-232 线接 X2 设了 Pr5.37=2(RS485) → 不通，应设 =1(RS232)
   坑3：~~Pr5.04 = 2 触发 Err38.0~~ 【作废】当时 Err38.0 的真实原因是 L1/L3 接线错误（即坑1），与 Pr5.04 无关。Pr5.04 是过行程禁止输入设定，含义待查
   坑4：默认校验是 8E1 不是 8N1
   坑5：试运转需要 S-RDY=1 + 控制模式正确 + Pr6.28=0
   坑6：参数初始化（AF_ini）会清空所有参数，包括 Modbus 配置
