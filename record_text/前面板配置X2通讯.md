前面板配置 X2 通讯指南
=======================

适用型号：MSMF022L1U2M + MADLT15SF（松下 MINAS A6 多功能型）

1. 目标
   通过驱动器前面板（5位LED + 5按键）修改参数，使 X2 端口启用 RS232 Modbus-RTU 通信。
   （硬件：USB-RS232 转换器 → X2 引脚 1/3/4，仅 1:1 调试）

2. 需要修改的参数（共 5 个）

   参数编号    名称                    设值   含义
   ──────── ──────────────────────── ───── ──────────────────────────────────────
   Pr5.37     Modbus 连接设置          1     Modbus-RTU（RS232、1:1）⭐ 最关键
   Pr5.31     轴编号（站号）            1     从站地址 = 1
   Pr5.29     RS232 通信波特率          2     9600 bps
   Pr5.38     Modbus 通信设置           0     偶校验 / 1停止位（出厂默认，不用改）
   Pr6.28     特殊功能选择              1     通过 Modbus 使 Block 动作有效 ⭐

   参数取值说明：

   Pr5.37 — Modbus 连接设置：
     0 = MINAS 标准协议（出厂值，即不启用 Modbus）
     1 = Modbus-RTU（RS232、仅 1:1）  ← 我们要的
     2 = Modbus-RTU（RS485、支持 1:N）

   Pr5.29 — RS232 通信波特率（RS485 模式时改用 Pr5.30）：
     0=2400, 1=4800, 2=9600, 3=19200, 4=38400, 5=57600, 6=115200, 7=230400

   Pr5.38 — Modbus 通信设置：
     0=偶校验/1停止位（出厂默认）
     1=偶校验/2停止位
     2=奇校验/1停止位
     3=奇校验/2停止位
     4=无校验/1停止位
     5=无校验/2停止位

   Pr6.28 — 特殊功能选择：
     0 = Block 动作无效（出厂值）
     1 = 通过 Modbus 通信使 Block 动作有效  ← 我们要的
     2 = 通过输入信号使 Block 动作有效
     4 = 通过输入信号使 Block 动作有效（脉冲列也有效）

3. 前面板按键

   ┌──────────────────────────────────┐
   │  ▮ ▮ ▮ ▮ ▮  ← 6位 LED 数码管    │
   │                                  │
   │   M    ◀    ▲    ▼    S         │
   │  MODE SHIFT UP  DOWN SET         │
   └──────────────────────────────────┘

   M (MODE)  — 切换模式
   ◀ (SHIFT) — 左移选位
   ▲ (UP)    — 数值/编号加
   ▼ (DOWN)  — 数值/编号减
   S (SET)   — 进入/退出/确认

4. 6 大模式（每按一次 M 切换）

   监视模式 → 参数设定 → EEPROM写入 → 辅助功能 → 复制功能 → 自动增益调整
   显示首字：  d-xx    Pr xx      EE_SEt    AF_xxx    CoPy      Au_xxx

5. 操作步骤

   ① 上电 → 默认进入监视模式，显示如 d00.00

   ② 按 M 一次 → 进入"参数设定模式"，显示 Pr0.00

   ③ 逐个修改 5 个参数：

   改每一个参数的操作：
   a. 用 ◀ 和 ▲▼ 把显示调到目标 Pr 编号（如 Pr 5.37）
   b. 按 S → 显示当前值
   c. 用 ◀ 和 ▲▼ 修改数值
   d. 按 S → 保存并返回 Pr 编号画面
   e. 继续改下一个参数

   推荐修改顺序：
   Pr5.37 → 改为 1   （先切协议，否则其他通信参数可能不生效）
   Pr5.31 → 改为 1   （站号）
   Pr5.29 → 改为 2   （9600 bps，RS232 专用波特率参数）
   Pr5.38 → 保持 0   （出厂默认偶校验/1停止位，确认即可）
   Pr6.28 → 改为 1   （启用 Modbus 控制 Block）

   ④ 所有参数改完后，按 M → 进入 "EEPROM 写入模式"，显示 EE_SEt

   ⑤ 按住 ▲ 约 3 秒，观察显示变化：
   EE_SEt → StArt → FInISH → rESEt
     StArt  = 开始写入
     FInISH = 写入完成
     rESEt  = 需要断电重启

   ⑥ 断电重启驱动器（控制电源 L1C/L2C 断开 5 秒以上再上电）

   ⑦ 重启后确认参数：重新进入参数设定模式，逐一查看 5 个参数是否正确

6. 修改后的 VOFA+ 串口设置

   波特率：9600
   数据位：8
   校验：  Even（偶校验）← 出厂默认 8E1，不是 8N1！
   停止位：1

7. X2 接线（RS232 模式，依据 sx-zsv00015 / 硬件接线.txt）

   USB-RS232 转换器 TXD → X2 引脚 4 (RXD，驱动器收)
   USB-RS232 转换器 RXD → X2 引脚 3 (TXD，驱动器发)
   USB-RS232 转换器 GND → X2 引脚 1 (GND)

   ⚠️ TXD/RXD 必须交叉接（PC的TX接驱动器的RX）。
   ⚠️ RS232 是 1:1 通信，最大距离约 15 米，不能挂多台。
   ⚠️ 引脚编号以驱动器外壳标注为准。

   （若日后要走 RS485：Pr5.37=2，波特率改用 Pr5.30，接 X2 引脚 8=A+/7=B-/1=GND）

8. 验证通信

   重启完成后，VOFA+ 打开串口，发送回环测试帧：
   01 08 00 00 12 34 ED 7C

   收到原样回发 → 通信成功！
   无回应 → 检查 A/B 线是否接反、串口参数是否为 9600/8E1

9. 常见问题

   现象               原因                    解决
   ───────────────── ──────────────────────── ──────────────────────
   显示 Loc           前面板被锁定              Pr5.35 设为 0 解锁
   EE 写入 Error      写入失败                  再试一次，确保不在报警中
   通信无回应         TXD/RXD 没有交叉接        互换 PC 侧 TX/RX 与驱动器侧
   通信无回应         波特率/校验不匹配          确认 VOFA+ 为 9600/8E1
   回环测试无回应     Pr5.37 仍为 0             确认已改为 1 并重启
   回环测试无回应     Pr5.37 设成 2 了          USB-232 线必须配 Pr5.37=1
   写线圈 0060h 无效  Pr6.28 仍为 0             确认已改为 1 并重启
