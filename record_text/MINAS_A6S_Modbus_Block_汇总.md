## 2. Modbus通信规格

| 项目 | 规格 |
|------|------|
| 协议 | MODBUS-RTU（仅限二进制模式） |
| 物理层 | RS232（1:1）、RS485（1:N，最大31轴） |
| 波特率 | 2400/4800/9600/19200/38400/57600/115200/230400 bps |
| 数据位 | 8 bit |
| 奇偶校验 | 无/偶数/奇数（出厂值：偶数） |
| 开始位 | 1 bit |
| 结束位 | 1/2 bit（出厂值：1 bit） |

### 波特率设置

| 设置值 | 波特率 |
|--------|--------|
| 0 | 2400 bps |
| 1 | 4800 bps |
| 2 | 9600 bps |
| 3 | 19200 bps |
| 4 | 38400 bps |
| 5 | 57600 bps |
| 6 | 115200 bps |
| 7 | 230400 bps |

- RS232波特率：Pr5.29
- RS485波特率：Pr5.30

### 奇偶/结束位设置 (Pr5.38)

| 设置值 | 奇偶 | 结束位 |
|--------|------|--------|
| 0 | Even | 1 bit |

## 3. 通信线路连接

### RS232（Pr5.37=1，1:1通信）

X2接口引脚：
- Pin 4: RXD ← 上位TXD
- Pin 3: TXD → 上位RXD
- Pin 1: GND — 上位GND

接口芯片：SN751701同等品

### RS485（Pr5.37=2，1:N通信，最大31轴）

X2接口引脚：
- Pin 6: RS485+
- Pin 5: RS485-
- Pin 1: GND
- Pin 8: RS485+（终端）
- Pin 7: RS485-（终端）

接口芯片：ADM485同等品

## 4. 通信时序

### 通常通信

```
T0: 停顿间隔时间（3.5个字符传输时间或0.75ms，取较长者）
T1: 命令处理时间（随命令变化）
T2: 数据返回等待时间（0~10000ms，Pr5.39设置，出厂值0ms）
```

- 普通命令处理时间：数ms
- 参数EEPROM写入（寄存器1020h写入6173h）：约300ms

### 广播通信（使用RS485时）

- 从站地址设为00h可进行广播
- 各从站根据Pr5.42设定响应动作

### 通信异常时

- 通信超时检出：Pr5.40设定，设置值为0时不检出Err80.0
- 连续接收错误：Pr7.90设定检测次数，0时异常检测无效

## 5. Modbus数据结构

### 发送命令（主站→从站）

| 字节 | 内容 |
|------|------|
| 从站地址 | 1~247（0为广播） |
| 功能码 | 01h/03h/05h/06h/08h/0Fh/10h |
| 数据 | 依功能码而定 |
| CRC-L | CRC校验低位 |
| CRC-H | CRC校验高位 |

### 响应（从站→主站）

- 正常响应：回送从站地址+功能码+数据+CRC
- 异常响应：功能码最高位置1 + 异常码1字节

## 6. 功能码

| 功能码 | 功能 | 内容 |
|--------|------|------|
| 01h | 读取线圈 | 读取线圈信息 |
| 03h | 读取寄存器 | 读取监视器、参数等寄存器 |
| 05h | 写入线圈 | 向线圈写入（ON:FF00h / OFF:0000h） |
| 06h | 写入寄存器 | 向寄存器写入（伺服参数请使用10h） |
| 08h | 通信诊断 | 回环测试（诊断子码0000h） |
| 0Fh | 写入复数线圈 | 向复数线圈写入 |
| 10h | 写入复数寄存器 | 向复数寄存器写入（推荐用于伺服参数） |

---

## 7. CRC校验

| 项目 | 规格 |
|------|------|
| 生成多项式 | CRC1-16（X^16 + X^15 + X^2 + 1） |
| 初始值 | FFFFh |
| 字节序 | 低字节序（低位字节在前，高位字节在后） |
| 错误处理 | CRC不一致时废弃接收数据，不返回报警 |

## 16. VOFA+控制操作指南

### 16.1 驱动器初始设置（必须通过前面板或PANATERM设定）

1. **Pr5.37 = 1** → Modbus-RTU(RS232通信)
2. **Pr5.29 = 6** → 波特率115200bps（与VOFA+一致）
3. **Pr5.38 = 4** → None奇偶 / 1bit结束位
4. **Pr6.28 = 1** → 通过Modbus通信使Block动作有效
5. **Pr5.31 = 1** → 轴编号1（Modbus从站地址=1）
6. 设定后**重启驱动器电源**

### 16.2 VOFA+ Modbus RTU连接设置

- 协议：Modbus RTU
- 串口：COM口（对应RS232连接）
- 波特率：115200
- 数据位：8
- 奇偶校验：None
- 停止位：1
- 从站地址：1（与Pr5.31一致）

### 16.3 基本控制流程（Modbus启动Block动作）

#### Step 1: 获取Modbus通信执行权

向寄存器 **4300h** 写入 **0055h**（获取执行权）

#### Step 2: 设定Block动作速度/加减速

| 寄存器 | 功能 | 示例值 |
|--------|------|--------|
| 4600h | 速度[0] | 500 (r/min) |
| 4610h | 加速时间[0] | 100 (ms) |
| 4620h | 减速时间[0] | 100 (ms) |

#### Step 3: 设定Block命令和数据

以JOG正方向为例（Block 0）：

**Block command 0** (4800h)：
- 字节3(命令码) = 03h (JOG)
- 字节2(数据引用1) = 00h (速度[0])
- 字节1(数据引用2=00h加速, 数据引用3=00h减速, 数据引用5 bit1=0 bit0=0)
- 字节0 bit4(数据引用4 bit0) = 0 (正方向)

**Block data 0** (4802h)：
- 数据引用7 = 0 (JOG不使用)

#### Step 4: 指定Block编号并启动

1. 向寄存器 **4414h** 写入 **0** (Block编号0)
2. 向线圈 **0120h** 写入 **FF00h** (STB=ON，启动Block动作)

#### Step 5: 停止Block动作

向线圈 **0124h** 写入 **FF00h** (S-STOP=ON，减速停止)

#### Step 6: 释放执行权

向寄存器 **4300h** 写入 **00AAh**（断开执行权）

### 16.4 伺服使能控制

| 操作 | Modbus命令 |
|------|-----------|
| 伺服ON | 向线圈0060h写入FF00h |
| 伺服OFF | 向线圈0060h写入0000h |
| 报警清除 | 向线圈0061h写入FF00h |

### 16.5 常用监视器读取

| 寄存器 | 名称 | 单位 | 用途 |
|--------|------|------|------|
| 4D06h | 实际速度 | r/min | 监视电机转速 |
| 4D08h | 转矩指令 | 0.05% | 监视负载 |
| 4001h | Error code | - | 读取报警代码 |
| 4D22h | 控制模式 | - | 确认当前控制模式 |
| 4413h | Block statusflag | - | Block动作状态 |
| 0140h | BUSY | - | Block动作执行中 |

### 16.6 EEPROM保存

向寄存器 **1020h** 写入 **6173h** → 执行EEPROM写入（约300ms）

### 16.7 报警清除

向寄存器 **4102h** 写入 **7274h** → 清除报警·警告
