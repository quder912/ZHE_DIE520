## 一、Block 在 Modbus 寄存器里的存储结构

每个 Block 占 **4 个 16-bit 寄存器 = 64 bit**：

```
Block N 地址 = 0x4800 + N × 4

Block 0:  4800h~4803h    Block command (32-bit) + Block data (32-bit)
Block 1:  4804h~4807h
...
Block 255: 4BFCh~4BFFh
```

| 寄存器偏移 | 名称 | 占用位 | 说明 |
|---|---|---|---|
| +0~+1 (4800h+4801h) | Block command | 高 32 bit | 指令字段：命令代码 + 数据引用1~6 |
| +2~+3 (4802h+4803h) | Block data    | 低 32 bit | 数据字段：数据引用7（位置脉冲数/Block号/比较值等） |

256 个 Block 占满 4800h~4BFFh（共 1024 个寄存器）。

## 二、64-bit Block 数据的位字段分布（所有命令统一）

按文档原文，**整条 64-bit 数据从高位（MSB=63）到低位（LSB=0）**的字段：

```
bit63 ────────────────────────────── bit0
┌──────┬───────┬───────┬───────┬───────┬───────┬───────┬─────────────┐
│ 命令 │ 数据  │ 数据  │ 数据  │ 数据  │ 数据  │ 数据  │   数据      │
│ 代码 │ 引用1 │ 引用2 │ 引用3 │ 引用4 │ 引用5 │ 引用6 │   引用7     │
│ 8bit │ 4bit  │ 4bit  │ 4bit  │ 2bit  │ 2bit  │ 8bit  │   32bit     │
└──────┴───────┴───────┴───────┴───────┴───────┴───────┴─────────────┘
 bit63 bit55   bit51   bit47   bit43   bit41   bit39   bit31 ─── bit0
~bit56  ~bit52  ~bit48 ~bit44  ~bit42  ~bit40  ~bit32
```

### 数据引用 5（Block 迁移条件）—— 所有命令共用

| Bit | 名称 | 取值 | 含义 |
|---|---|---|---|
| **bit1 (MSB)** | **处理继续** | 0 | **本 Block 结束**（无视 bit0，整条 Block 序列停止） |
| | | 1 | Block 动作继续 |
| **bit0 (LSB)** | **下一个 Block 启动** | 0 | 本 Block 启动后立即向下一个 Block (N+1) 迁移 |
| | | 1 | 本 Block 完成后向下一个 Block (N+1) 迁移 |

**关键真值表**：

| bit1 (MSB) | bit0 (LSB) | 二进制 | 实际行为 |
|:---:|:---:|:---:|---|
| 0 | × | `0X` | **结束**（Block 动作终止，BUSY 变 0） |
| 1 | 0 | `10` | 启动后立刻执行 N+1（并行/合成动作） |
| 1 | 1 | `11` | 完成后执行 N+1（连续动作，最常用） |

> 注：Jump 命令 (9h) 不按这个规则——启动后强制跳到指定 Block，与 LSB 无关。

## 三、所有命令代码（命令代码 = bit56~bit63 = 高字节）

| 代码 | 命令 | 说明 |
|---|---|---|
| **01h** | **相对位置定位** | 走相对距离（pulse） |
| **02h** | **绝对位置定位** | 走到绝对位置（pulse） |
| **03h** | **JOG** | 固定速度持续运转（需 S-STOP 停止） |
| **04h** | **原点复位** | Home 回归 |
| **05h** | **减速停止** | 强制停止当前定位 |
| **06h** | **速度更新** | 动作中改速度 |
| **07h** | **减法计数器启动** | 计数器倒计时（用于延时/条件） |
| **08h** | **输出信号操作** | 操作 B-CTRL1~6 输出 |
| **09h** | **跳转 (Jump)** | 跳到指定 Block 号 |
| **0Ah** | **条件转移 (=)** | 满足"等于"条件则跳转 |
| **0Bh** | **条件转移 (>)** | 满足"大于"则跳转 |
| **0Ch** | **条件转移 (<)** | 满足"小于"则跳转 |

> **没有"循环 N 次"内建字段。** 循环只能用 Jump 形成无限循环 + 外部停止（S-STOP 或条件转移命令）退出。
> **没有"正反转"复合命令。** 正反往复必须用 2 个相对定位 Block + Jump 串联实现。

## 四、各命令字段详解

### 4-1 命令 01h：相对位置定位

| 数据引用 | 字段 | 取值 | 含义 |
|---|---|---|---|
| **代码** | 命令代码 | 01h | 固定 |
| **引用1** | 目标速度编号 | 0~15 | 引用 V0~V15（Pr60.00~Pr60.15）|
| **引用2** | 加速时间编号 | 0~15 | 引用 A0~A15（Pr60.16~Pr60.31）|
| **引用3** | 减速时间编号 | 0~15 | 引用 D0~D15（Pr60.32~Pr60.47）|
| **引用4** | — | — | 不使用 |
| **引用5** | Block 迁移条件 | 见上 | bit1=处理继续, bit0=下一个Block启动 |
| **引用6** | — | — | 不使用 |
| **引用7** | 相对移动量 | -2147483647~2147483647 | 指令单位（脉冲），带符号 |

| 数据引用 | 字段 | 取值 |
|---|---|---|

| 2 | 位置偏差 | pulse | 指令-当前 |
| 3 | 指令速度 | r/min | 滤波器前的速度指令 |
| 4 | 电机速度 | r/min | 电机当前速度 |
| 5 | 转矩指令 | 0.1% | 给电机的转矩指令 |
| 6 | 减法计数器 | — | 计数器当前值 |
| 7 | 输入信号 | — | **仅 0Ah(=) 可用** |
| 8 | 输出信号 | — | **仅 0Ah(=) 可用** |

## 五、速度/加减速档位寄存器

| 寄存器地址 | 名称 | 单位 | 范围 | Pr 编号 |
|---|---|---|---|---|
| 4600h~460Fh | Block velocity 0~15 (V0~V15) | r/min | 0~20000 | Pr60.00~Pr60.15 |
| 4610h~461Fh | Block acceleration 0~15 (A0~A15) | ms/(3000r/min) | 0~10000 | Pr60.16~Pr60.31 |
| 4620h~462Fh | Block deceleration 0~15 (D0~D15) | ms/(3000r/min) | 0~10000 | Pr60.32~Pr60.47 |

**Block 内只填档位编号 0~15**，实际数值由这些独立寄存器维护。

## 六、Block 控制寄存器/线圈

| 地址 | 名称 | 属性 | 说明 |
|---|---|---|---|
| **4414h** | Block number | R/W | 写入要启动的 Block 编号 |
| **4413h bit0** | B_B_BUSY | R | 1=Block 运行中, 0=空闲 |
| **0120h** | STB（选通输入）| R/W | FF00h=启动 Block, 0000h=复位 |
| 0123h | H-STOP（立即停止）| R/W | FF00h 触发 |
| 0124h | S-STOP（减速停止）| R/W | FF00h 触发，**唯一能停 JOG/无限循环的方法** |

启动流程：
```
1. 写 4414h = N        指定 Block 编号
2. 写线圈 0120h = FF00 STB ON 触发
3. 轮询 4413h bit0     等待 BUSY → 0
4. 写线圈 0120h = 0000 STB 复位（除非 Pr5.42 bit2=1 自动复位）
```

## 八、构造一条 64-bit Block 数据的方法（示例）

**目标**：相对定位转 1 圈（假设 10000 pulse/圈），速度用 V0，加减速用 A0/D0，迁移条件 = 结束（独立动作）

### 字段填写

```
命令代码  = 0x01    （相对位置定位）
引用1   = 0x0     （V0 速度档位）
引用2   = 0x0     （A0 加速档位）
引用3   = 0x0     （D0 减速档位）
引用4   = 0b00    （不用）
引用5   = 0b00    （MSB=0 结束，LSB=任意）
引用6   = 0x00    （不用）
引用7   = 0x00002710 （10000 = 1 圈，正方向）
```

### 拼成 64-bit

按位顺序（高→低）：
```
8bit  4b  4b  4b  2b  2b  8b   32b
01h   0   0   0   00  00  00h  0000_2710h

二进制：
00000001 0000 0000 0000 00 00 00000000 00000000000000000010011100010000
```

### 分成 Block command (32-bit 高位) + Block data (32-bit 低位)

```
Block command = 0x01_0_0_0_0_0_00 = 0x01000000
Block data    = 0x00002710

寄存器：
4800h = 0x0100   (command 高 16)
4801h = 0x0000   (command 低 16)
4802h = 0x0000   (data 高 16)
4803h = 0x2710   (data 低 16)
```

### Modbus 写多寄存器帧（功能码 0x10）

写 Block 0 整条（4800h~4803h，4 个寄存器，8 字节数据）：

```
01 10 48 00 00 04 08  01 00 00 00 00 00 27 10  [CRC_L CRC_H]
站 功 起址    数   字节  数据(8字节)            CRC
```

实际 CRC 算出后补上即可（用我们之前的 Python crc16 函数）。

> 后续 C# 实现 Block 编辑器时，按本文档的字段定义封装即可。

## 九、对照旧记录的修正

❌ 旧记录："每个 Block 占 16 个 16-bit 寄存器，Block 1 = 4810h"
✅ 实情：每个 Block 占 **4 个** 16-bit 寄存器，**Block 1 = 4804h**

❌ 旧记录："Block 有'下一个 Block 号'字段在偏移 +6"
✅ 实情：默认下一个 Block = N+1，要跳到任意编号必须用 Jump (9h)

❌ 旧记录："Err 55.0 可能是 Block 表为空"
✅ 实情：Err 55.0 是全闭环外部传感器接线/配置异常，与 Block 无关

❌ 旧记录："Block 字段表格式未知，需要 PANATERM 配置后反推"
✅ 实情：所有字段定义在 sx-zsv00015 P83~P95 已完整提取
