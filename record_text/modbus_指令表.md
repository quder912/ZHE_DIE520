# 松下 MINAS A6 伺服驱动器 Modbus 指令参考

基于 SX-DSV03054（DSV系列）和 SX-ZSV00015（A6 标准系列）技术手册整理。

---

## 1. 通信规格

| 项目 | 规格 |
|------|------|
| 物理层 | RS485/RS232（X2 连接器）、USB（X1 连接器，虚拟串口） |
| 协议 | Modbus RTU |
| 数据格式 | 8bit、无校验/奇校验/偶校验、1stop/2stop |
| 波特率 | 9600 / 19200 / 38400 / 57600 / 115200 bps |
| 站号范围 | 1~31（Pr5.00 设定，默认 1） |

默认参数（出厂值）：9600 bps、8N1、站号 1。

---

## 2. 功能码一览

| 功能码 | 名称 | 说明 |
|--------|------|------|
| 01h | 读取线圈 (Read Coils) | 读取 ON/OFF 状态（S-RDY、报警、到位等） |
| 03h | 读取寄存器 (Read Holding Registers) | 读取参数、监视值、位置/速度/转矩反馈 |
| 05h | 写入线圈 (Write Single Coil) | ON/OFF 控制（伺服使能、STB 触发、急停等） |
| 06h | 写入寄存器 (Write Single Register) | 单寄存器写入（推荐用 10h 写参数） |
| 08h | 通信诊断 (Diagnostic) | 回环测试（Loopback Test） |
| 0Fh | 写入复数线圈 (Write Multiple Coils) | 批量 ON/OFF 操作 |
| 10h | 写入复数寄存器 (Write Multiple Registers) | 写参数的推荐指令，写 Block 动作数据 |

## 3. 帧格式

### 发送帧（主机 → 驱动器）

| 字节 | 内容 |
|------|------|
| 1 | 从站地址（站号） |
| 2 | 功能码 |
| N | 数据 |
| 2 | CRC16 校验（低位 + 高位） |

### 正常响应（驱动器 → 主机）

| 字节 | 内容 |
|------|------|
| 1 | 从站地址（站号） |
| 2 | 功能码（与请求相同） |
| N | 返回数据 |
| 2 | CRC16 校验 |

### 异常响应（功能码最高位=1）

| 字节 | 内容 |
|------|------|
| 1 | 从站地址 |
| 2 | 功能码 | 80h（示例：03h → 83h） |
| 1 | 例外码 |
| 2 | CRC16 校验 |

**例外码：**

| 码 | 含义 |
|----|------|
| 01h | 功能码异常（不支持的功能码） |
| 02h | 数据地址异常（地址越界） |
| 03h | 数据异常（写入值非法） |
| 04h | 响应处理异常（驱动器中处理失败） |

## 7. Block动作完整流程

```
1. 设置Pr6.28=1         → 选择Modbus通信启动Block
2. 写线圈0060h=FF00h    → 伺服使能ON
3. 写寄存器4414h=N       → 指定Block号N
4. 写线圈0120h=FF00h    → STB启动Block
5. 读寄存器4413h         → 监视bit0(B_BUSY)=0为完成
```

## 8. CRC16 校验（Modbus RTU 标准）

多项式：x^16 + x^15 + x^2 + 1 (0x8005)
初始值：0xFFFF
计算结果低位先发送。

参考C#实现：

```csharp
private static ushort CRC16(byte[] data, int start, int len)
{
    ushort crc = 0xFFFF;
    for (int i = start; i < start + len; i++)
    {
        crc ^= data[i];
        for (int j = 0; j < 8; j++)
        {
            if ((crc & 0x0001) != 0)
                crc = (ushort)((crc >> 1) ^ 0xA001);
            else
                crc >>= 1;
        }
    }
    return crc;
}
```

使用示例：`CRC16(new byte[] { 0x01, 0x05, 0x00, 0x60, 0xFF, 0x00 }, 0, 6)` 返回 `0x2788`，帧中发送顺序为 `88 27`。
