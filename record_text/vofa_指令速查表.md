# VOFA+ Modbus 指令速查表（松下 A6 伺服）

> 站号 = 1，所有帧 CRC 已计算好，可直接在 VOFA+ HEX 模式下复制粘贴发送。
>
> 使用前提：
> - VOFA+ 数据源 = SerialPort，协议解析器 = RawData
> - 发送区选 **HEX 模式**
> - 串口参数与驱动器一致（出厂默认 **9600 / 8E1**（偶校验/1停止位）/ 站号 1）
>
> ⚠️ **首次使用必须先用前面板设置以下参数**（详见 `前面板配置X2通讯.md`，如已配置请跳过）：
> - Pr5.37 = 1 （启用 Modbus-RTU via **RS232**，USB-RS232 转接线模式）
> - Pr5.31 = 1 （站号 = 1）
> - Pr5.29 = 2 （9600 bps，RS232 专用波特率参数；走 RS485 时改用 Pr5.30）
> - Pr5.38 = 0 （偶校验/1停止位，出厂默认）
> - Pr6.28 = 1 （Modbus 控制 Block 动作有效）
>
> 不设这些参数，驱动器根本不会响应 Modbus 帧，VOFA+ 一直收不到回应。
>
> **X2 接线（RS232 模式）**：USB-232 的 TXD↔X2 引脚4(RXD)，RXD↔X2 引脚3(TXD)，GND↔X2 引脚1。
> TXD/RXD 必须交叉。

## 一、通信测试（首先做这步）

| 用途 | 发送帧（HEX） | 期望响应 |
|---|---|---|
| 回环测试 (Loopback) | `01 08 00 00 12 34 ED 7C` | 原样返回 |
| 回环测试 (备用数据) | `01 08 00 00 AA 55 5E 94` | 原样返回 |

**收到原样回发 → 通信链路 OK**，可继续下面。

## 二、参数配置（已通过前面板完成则跳过）

通过 Modbus 写参数的指令（需要先把 Pr5.37 设好，否则通信都不通）。
具体寄存器地址要查 **sx-zsv00015** 的"参数 ↔ 寄存器映射表"，PDF 文档目前没完整提取，先以前面板修改为主。

⚠️ 通过 Modbus 写参数后，要把值保存到 EEPROM 需要额外指令：

| 操作 | 发送帧 |
|---|---|
| 保存所有参数到 EEPROM (1020h ← 6173h) | `01 06 10 20 61 73 4F BF` |

参数 EEPROM 写入约耗时 **300 ms**，写入期间不要发送其他命令。

写入完成后**断电重启**驱动器，部分参数才生效。

## 六、状态寄存器读取（最常用）

| 用途 | 发送帧 | 响应说明 |
|---|---|---|
| **读 Statusword** | `01 03 40 00 00 01 91 CA` | 驱动器主状态字 |
| **读 错误码 Error code** | `01 03 40 01 00 01 C0 0A` | 0 = 正常，非 0 = 报警 |
| **读 Block 状态 (B_BUSY)** | `01 03 44 13 00 01 61 3F` | bit0 = 1 表示 Block 运行中 |
| 读 Block 控制监视 | `01 03 44 12 00 01 30 FF` | STB / HOME / STOP 状态映射 |
| 读 电机动作状态 | `01 03 44 10 00 01 91 3F` | 转矩限制 / 定位完成等 |
| **读 电机实际速度** | `01 03 40 25 00 01 80 01` | 单位 r/min |
| 读 指令速度 | `01 03 40 1E 00 01 F1 CC` | 单位 r/min |
| 读 位置偏差 (32bit) | `01 03 40 22 00 02 71 C1` | 单位 pulse |
| 读 电机负载率 | `01 03 40 0D 00 01 00 09` | 单位 % |
| 读 驱动器温度 | `01 03 40 12 00 01 31 CF` | 单位 ℃ |
| **读 编码器单圈位置** | `01 03 4D 24 00 02 93 6C` | 32bit |
| 读 绝对式多圈数据 | `01 03 4D 28 00 04 D3 6D` | 64bit |

## 推荐测试流程

```
1. 通信测试       → 回环测试帧
2. 读驱动器信息    → 确认型号
3. 读 Error code  → 确认无报警（应返回 0）
4. 伺服 ON
5. 读 Statusword  → 确认伺服已 ON
6. 指定 Block 号
7. STB ON 启动 Block
8. 循环读 4413h B_BUSY → 等待 bit0 = 0（完成）
9. 伺服 OFF
```

## 异常响应识别

如果返回的**功能码 = 原码 + 0x80**，说明出错：

| 异常功能码 | 含义 |
|---|---|
| `0x81` | 01h 读线圈异常 |
| `0x83` | 03h 读寄存器异常 |
| `0x85` | 05h 写线圈异常 |
| `0x86` | 06h 写寄存器异常 |
| `0x90` | 10h 写多寄存器异常 |

后面跟 1 字节异常码：

| 异常码 | 含义 |
|---|---|
| `01` | 功能码不支持 |
| `02` | 地址越界 |
| `03` | 数据值非法 |
| `04` | 从设备故障 |

## 附：CRC16 自算脚本（Python）

```python
def crc16(data: bytes) -> bytes:
    """松下 A6 / Modbus RTU 标准 CRC16，返回小端 2 字节"""
    crc = 0xFFFF
    for b in data:
        crc ^= b
        for _ in range(8):
            if crc & 1:
                crc = (crc >> 1) ^ 0xA001
            else:
                crc >>= 1
    return bytes([crc & 0xFF, (crc >> 8) & 0xFF])

def build(hex_str: str) -> str:
    data = bytes.fromhex(hex_str.replace(" ", ""))
    full = data + crc16(data)
    return " ".join(f"{b:02X}" for b in full)

# 用法：
print(build("01 06 44 14 00 0A"))   # → 指定 Block 号 = 10
```

### 配置参数（依据 sx-zsv00015 权威文档）

| Pr 编号 | 名称 | 取值 |
|---|---|---|
| **Pr5.29** | RS232 通信波特率 | 0=2400 ... 6=115200, 7=230400 |
| **Pr5.30** | RS485 通信波特率 | 0=2400, 1=4800, 2=9600, 3=19200, 4=38400, 5=57600, 6=115200, 7=230400 |
| **Pr5.31** | 轴编号（站号） | 1~31 |
| **Pr5.37** | Modbus 连接设置 ⭐ | 0=MINAS标准, 1=Modbus RS232, **2=Modbus RS485** |
| **Pr5.38** | Modbus 通信设置 | 0=Even/1bit (默认), 1=Even/2bit, 2=Odd/1bit, 3=Odd/2bit, 4=None/1bit, 5=None/2bit |
| **Pr5.39** | Modbus 返回等待时间 | 0~10000 ms |
| **Pr5.40** | Modbus 通信超时时间 | 0~10000 ms (0=不检测) |
| **Pr5.42** | Modbus 广播设置 | bit 配置 |
| **Pr6.28** | 特殊功能选择 ⭐ | 0=无效, **1=Modbus 启动 Block**, 2/4=I/F信号启动 |

> ⚠️ 各 Pr 参数对应的 Modbus 寄存器地址需要查 sx-zsv00015 完整文档（PDF 中），目前 txt 提取不完整，建议**通过前面板修改**而非 Modbus 在线写入。
